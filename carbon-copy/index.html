About Me

I’m a student majoring in Electro-Mechanical Engineering with hands-on experience building AI-powered systems that bridge hardware control and software. I focus on practical, end-to-end projects—from embedded devices to web backends and interactive UIs.

AI & IoT Projects

  • Voice-Driven Accessibility Kiosk — Built with Python and Flask, integrating OpenAI Whisper (STT), NFC-based allergy profiles, and a shopping cart workflow. Implemented continuous speech handling, noise filtering, and threaded TTS control.
  • Rescue/Companion Robot (WALL-E scale) — Raspberry Pi 5 + ESP32 control stack, caterpillar tracks, real-time video via Flask, and person detection with models converted to TensorFlow Lite. Delivers aid on detection and confirms status via voice prompts.
  • AI Virtual Streamer — LangChain + GPT responses, emotion-driven facial expressions, and ElevenLabs TTS. Designed for smooth real-time chat processing and logging.

Software Development

  • Python (3.10/3.11), Flask/FastAPI, JavaScript; real-time features with WebSockets and threading.
  • Computer vision with OpenVINO and TFLite; dataset training on Ultralytics HUB; deployment on Pi.
  • Clean, accessible front-ends (semantic HTML/CSS) and RESTful APIs with JWT/session handling.

Hardware & Networking

  • PLC: LS XGK series with XG5000; basic HMI work; sensor/actuator integration.
  • ESP32 motor control via relays; NFC and ultrasonic sensor pipelines to Python backends.
  • Secure networking labs: OpenVPN, SoftEther, WireGuard; tunneling, routing, and NAT rules on Raspberry Pi.

What I’m Aiming For

Building human-centered robotics and AI systems that are reliable, explainable, and deployable on real hardware. Always iterating, measuring, and shipping.

Interruptible TTS & Always‑On STT in a Kiosk

Implemented continuous mic capture with Whisper STT, ending on silence with energy thresholds and debounce. During TTS playback, the mic closes and re‑opens on completion; if user speech arrives mid‑TTS, playback is interrupted via a thread‑safe flag. Result: natural turn‑taking and fewer false barge‑ins.
